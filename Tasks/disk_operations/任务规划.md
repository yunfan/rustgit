# 任务规划：为 rustgit 库增加完整落盘（Disk Persist）操作支持

## 任务目标
当前 `rustgit` 库主要基于纯内存和网络流实现数据结构的流转，缺少对原生 `.git` 目录结构的直接写入和读取支持。
本任务旨在将其扩充为一个全功能的 Git 本地库，使其能够执行实体化的读写操作（落盘操作），从而真正接管和管理宿主系统的物理 Git 仓库。

## 需求说明
- **存储引擎抽象化 (Storage Abstraction)**：对现有完全基于内存的 `ObjectStore` 核心数据流进行重构，抽象出一个统一的 `StorageBackend` trait，使 `Repository` 解耦出对特定存储层的依赖。
- **内存存储实现 (MemoryStorage)**：保留并平移现有的 `ObjectStore`（基于 `LiteMap` 等）作为纯内存的存储驱动，保障兼容性。
- **Git 兼容磁盘存储 (GitDiskStorage)**：增加原生的 `.git` 物理目录操作和标准 Git 的 zlib 压缩松散对象读写，使其能完全对接并管理外部标准 Git 工具初始化的仓库。
- **自研 KV 数据库优化存储 (KvDatabaseStorage)**：设计并预留基于底层 KV 引擎（如 sled / RocksDB 等）直接读写和寻址的高性能存储方案。
