# 实施计划：为 rustgit 库增加多端可替换存储目标支持

## 阶段 1: 存储系统架构重构 (Storage Abstraction)

**目标**: 将底层的存储行为从 `Repository` 中解耦，抽象为可替换的接口。
**成功标准**: 提取出一个或多个类似 `trait StorageBackend` 的定义，并完成内存驱动 `MemoryStorage` 对当前 `LiteMap` 逻辑的无缝平移。
**测试用例**: 原有的内存测试用例运行无误。
**状态**: 未开始

## 阶段 2: 实现标准 Git 兼容的磁盘存储 (GitDiskStorage)

**目标**: 为 `StorageBackend` 增加基于标准 `.git` 目录和 zlib 松散文件压缩的物理实现。
**成功标准**: 提供 `GitDiskStorage`，能准确读写 `.git/objects` 和解析 `.git/refs` 文件。
**测试用例**: 通过写入对象后，使用外部的原生 Git CLI (`git cat-file -p`) 能直接解析我们的文件。
**状态**: 未开始

## 阶段 3: 实现基于 KV 数据库优化的存储 (KvDatabaseStorage)

**目标**: 新增使用 KV 引擎存储散列特征的后端支持。
**成功标准**: 能够创建 `KvDatabaseStorage`，完成键（Hash）对值（序列化对象）的快速插拔支持。
**测试用例**: 提供基于该 KV 结构的 Benchmark 或在 `rustgit-cmd` 指定初始化引擎时的正确行为验证。
**状态**: 未开始

## 阶段 4: 前端调度器集成设计

**目标**: 在 `Repository::new()` / `init()` 等上层初始化 API 暴露对存储后端的动态分发能力。
**成功标准**: 可通过类似 `Repository::with_storage(GitDiskStorage::new(path))` 初始化不同实例。
**状态**: 未开始
